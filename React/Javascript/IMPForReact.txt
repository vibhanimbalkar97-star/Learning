Array Methods (Highest Priority)
✅ map() = Purpose: Transform every item in an array.
✅ filter() = Returns only matching items.
✅ find() = Returns only the first matching item.
✅ findIndex() =  returns the index of the first element in an array that satisfies a given testing condition, returning -1 if no match is found.
✅ reduce() = reduces an array to a single value by executing a callback function on each element.
✅ some()=Returns true if at least one element matches. gives false and true result.
✅ every() = Returns true if all satisfy condition.
✅ sort() = sort array in ascending order
Why are a and b taken?

The sort() method automatically picks two elements from the array and passes them to your callback function.
a = first element being compared
b = second element being compared
You don't call the function yourself. JavaScript does it internally.

✅ slice() = Doesn't change original array.
splice() = change original array. Gives middle element.
✅ includes()= checks if an array or a string contains a specific value. It returns true if the value is found, and false if it is not.
✅ forEach()=iterate through array but not return value.
  indexOf()=method finds the first position of a value in an array or string and returns its 0-based index, or -1 if the value is not found.

Easy rule to remember

| Data Type            | Best Method                                            |
| -------------------- | ------------------------------------------------------ |
| **Array of objects** | `reduce()`, `map()`, `filter()`, `find()`              |
| **Single object**    | `Object.keys()`, `Object.values()`, `Object.entries()` |

Interview Tip
Use Object.keys() when you want to work with an object's property names.
Use Object.values() when you want to calculate or iterate over an object's values.
Use reduce() directly on an array of objects when you need totals, averages, maximums, or other aggregate values.

1. Optional Chaining (?.)
Purpose

Safely access a property that might not exist.

Without ?., JavaScript can throw an error.

const user = {
  name: "John",
};

console.log(user.address.city); // Cannot read properties of undefined (reading 'city')

why? user.address = undefined
the js tries to read = undefined.city which is impossible

solution
const user = {
  name: "John",
};

console.log(user.address?.city);//no error
undefined

means:

If address exists, then access city.

Otherwise return undefined.

2. Nullish Coalescing (??)
Purpose

Provide a default value only when the value is:

null
undefined

const name = undefined;

console.log(name ?? "Guest"); // Guest

const name = "John";

console.log(name ?? "Guest"); //John

How it works
value ?? defaultValue

means

If value is null or undefined,
use defaultValue.

Otherwise use value.

| Operator | Meaning                                 | Example                | Output                                           |
| -------- | --------------------------------------- | ---------------------- | ------------------------------------------------ |
| `?.`     | Safely access a property                | `user.address?.city`   | `undefined` if `address` doesn't exist           |
| `??`     | Use a default for `null` or `undefined` | `user.name ?? "Guest"` | `"Guest"` only when `name` is `null`/`undefined` |


Quick memory trick
?. = "Does it exist?" → Safe property access.
?? = "If it's missing (null/undefined), use this instead." → Default value.
