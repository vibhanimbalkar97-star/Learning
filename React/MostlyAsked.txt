# 1. What is useState?

### Answer

`useState` is a React Hook that allows functional components to store and update state.

### Syntax

```jsx
const [count, setCount] = useState(0);
```

* `count` → current state value
* `setCount` → function to update the state
* `0` → initial value

### Interview Answer

> `useState` is a React Hook used to create and manage state inside functional components. Whenever the state changes, React re-renders the component with the updated value.

---

# 2. Why do we use useState?

### Answer

We use `useState` when data changes over time and the UI should update automatically.

### Example

```jsx
const [count, setCount] = useState(0);
```

When

```jsx
setCount(5);
```

React updates the UI automatically.

### Interview Answer

> We use useState to manage dynamic data like counters, forms, API responses, toggles, and any value that changes and should update the UI.

---

# 3. What happens when state changes?

Example

```jsx
const [count, setCount] = useState(0);

setCount(1);
```

React

1. Updates the state.
2. Re-renders the component.
3. Updates only the changed part of the DOM (Virtual DOM reconciliation).

### Interview Answer

> When state changes, React schedules a re-render of the component, compares the new Virtual DOM with the previous one, and updates only the necessary parts of the real DOM.

---

# 4. Why don't we update state directly?

❌ Wrong

```jsx
count = count + 1;
```

React doesn't know the value changed.

Correct

```jsx
setCount(count + 1);
```

### Why?

Because React tracks changes through the setter function.

### Interview Answer

> We should never update state directly because React won't detect the change. We must use the setter function so React can update the state and re-render the component.

---

# 5. Why is state immutable?

Example

❌ Wrong

```jsx
user.name = "Rahul";
```

Correct

```jsx
setUser({
  ...user,
  name: "Rahul",
});
```

### Why?

React compares references.

If reference doesn't change,

React assumes nothing changed.

### Interview Answer

> React state should be treated as immutable. Instead of modifying the existing object or array, we create a new one so React can detect the change using reference comparison.

---

# 6. Why is state update asynchronous?

Example

```jsx
setCount(count + 1);

console.log(count);
```

Output

```text
0
```

not

```text
1
```

### Why?

React batches state updates for better performance.

### Interview Answer

> State updates are asynchronous because React batches multiple updates together to improve performance. Therefore, the updated value is available in the next render, not immediately after calling the setter.

---

# 7. What is batching?

Example

```jsx
setCount(1);
setName("Rahul");
setAge(25);
```

React combines them into one render.

Instead of

```text
Render
Render
Render
```

React does

```text
Single Render
```

### Interview Answer

> React batches multiple state updates into a single render to improve performance and reduce unnecessary re-renders.

---

# 8. Difference between

```jsx
setCount(count + 1);
```

and

```jsx
setCount(prev => prev + 1);
```

### First

```jsx
setCount(count + 1);
setCount(count + 1);
```

Current count = 0

Both become

```jsx
setCount(1);
setCount(1);
```

Final value

```text
1
```

---

### Second

```jsx
setCount(prev => prev + 1);
setCount(prev => prev + 1);
```

Works like

```
0 → 1 → 2
```

Final

```text
2
```

### Interview Answer

> `setCount(count + 1)` uses the current render's value and can become stale if multiple updates happen together. `setCount(prev => prev + 1)` always uses the latest state, so it's the preferred approach when the new state depends on the previous state.

---

# 9. When should we use functional updates?

Use

```jsx
setCount(prev => prev + 1);
```

when

* Incrementing counter
* Decrementing
* Like button
* Cart quantity
* Toggle based on previous state
* Any update depending on the previous value

### Interview Answer

> Functional updates should be used whenever the new state depends on the previous state, because they always receive the latest value.

# 12. Can we have multiple useState?

Yes.

```jsx
const [name, setName] = useState("");
const [age, setAge] = useState(20);
const [city, setCity] = useState("");
```

### Interview Answer

> Yes, a component can have multiple `useState` hooks. Each hook manages an independent piece of state, which often makes the code simpler and easier to maintain.

---

# 13. When should we use one state object vs multiple states?

**Use multiple states** when the values are independent:

```jsx
const [name, setName] = useState("");
const [loading, setLoading] = useState(false);
```

**Use one object** when the values belong together:

```jsx
const [form, setForm] = useState({
  name: "",
  email: "",
});
```

### Interview Answer

> Use separate state variables for unrelated data, and use a single object when multiple fields represent the same entity, such as a form.

---

# 14. Does updating state always re-render the component?

Usually **yes**, but if you set the state to the **same value**, React may skip the re-render.

Example:

```jsx
const [count, setCount] = useState(5);

setCount(5);
```

React compares the old and new values using `Object.is()`. If they are equal, React avoids an unnecessary re-render.

### Interview Answer

> State updates normally trigger a re-render. However, if the new state is the same as the current state, React may skip the re-render as an optimization.

---

# 15. Can we call useState inside loops or conditions?

**No.**

❌ Wrong

```jsx
if (isLoggedIn) {
  const [name, setName] = useState("");
}
```

✅ Correct

```jsx
const [name, setName] = useState("");

if (isLoggedIn) {
  // use the state here
}
```

### Why?

Hooks must always be called in the same order on every render.

### Interview Answer

> No. Hooks must be called only at the top level of a component or a custom Hook. Calling them inside loops, conditions, or nested functions breaks React's Hook rules.

# 1. What is Event Handling in React?

### Answer

Event handling in React is the process of responding to user actions such as clicks, typing, form submission, mouse movements, and keyboard events.

Example:

```jsx
function App() {
  const handleClick = () => {
    alert("Button Clicked");
  };

  return <button onClick={handleClick}>Click</button>;
}
```

**Interview Answer:**

> Event handling in React allows components to respond to user interactions. React uses camelCase event names like `onClick` and event handlers are passed as functions.

---

# 2. How is Event Handling different in React compared to HTML?

### HTML

```html
<button onclick="save()">Save</button>
```

### React

```jsx
<button onClick={save}>Save</button>
```

### Differences

| HTML        | React         |
| ----------- | ------------- |
| onclick     | onClick       |
| Uses string | Uses function |
| Lowercase   | CamelCase     |

**Interview Answer:**

> In React, event names use camelCase (e.g., `onClick`) and event handlers are passed as functions instead of strings.

---

# 3. Why do we write `onClick={handleClick}` instead of `onClick={handleClick()}`?

### Correct

```jsx
<button onClick={handleClick}>Click</button>
```

The function runs **only when the button is clicked.**

---

### Wrong

```jsx
<button onClick={handleClick()}>Click</button>
```

The function runs **immediately during rendering**, not on click.

---

### Interview Answer

> `onClick={handleClick}` passes the function reference, so React calls it only when the event occurs. `onClick={handleClick()}` executes the function immediately during rendering.

---

# 4. How do you pass arguments to an event handler?

Example

```jsx
const greet = (name) => {
  alert(name);
};

<button onClick={() => greet("John")}>
  Click
</button>
```

### Why arrow function?

Without it

```jsx
<button onClick={greet("John")}>
```

The function executes immediately.

### Interview Answer

> To pass arguments, wrap the function call inside an arrow function so it executes only when the event occurs.

---

# 5. What is Synthetic Event in React?

### Answer

React wraps browser events inside a **SyntheticEvent**.

It provides a consistent event interface across all browsers.

Example

```jsx
const handleClick = (event) => {
  console.log(event.target);
};
```

### Benefits

* Cross-browser compatibility
* Same API everywhere
* Better performance

### Interview Answer

> React uses SyntheticEvent, which is a wrapper around the native browser event. It provides a consistent API across different browsers.

---

# 6. What is `event.preventDefault()`?

Example

```jsx
const handleSubmit = (e) => {
  e.preventDefault();
  console.log("Submitted");
};
```

Without it,

the page reloads after form submission.

### Interview Answer

> `event.preventDefault()` prevents the browser's default behavior, such as refreshing the page when a form is submitted.

# 7. What is `event.stopPropagation()`?

Suppose

```jsx
<div onClick={parentClick}>
    <button onClick={childClick}>
        Click
    </button>
</div>
```

Normally

```
Button Click

↓

Child Click

↓

Parent Click
```

Using

```jsx
const childClick = (e) => {
    e.stopPropagation();
};
```

Parent won't execute.

### Interview Answer

> `stopPropagation()` prevents an event from bubbling up to parent elements.

# 8. What is Event Bubbling?

Example

```
Grandparent

↓

Parent

↓

Child Button
```

Click Child

Execution

```
Child

↓

Parent

↓

Grandparent
```

This is Event Bubbling.

### Interview Answer

> Event bubbling means an event starts from the target element and propagates upward through its parent elements.

# 9. What is Event Capturing?

Order

```
Grandparent

↓

Parent

↓

Child
```

Execution

```
Grandparent

↓

Parent

↓

Child
```

React supports capturing using

```jsx

```

### Interview Answer

> Event capturing is the opposite of bubbling. The event travels from the outermost parent to the target element.

# 12. What is the difference between `onClick` and `onDoubleClick`?

```jsx
<button onClick={click}>
```

Runs once.

```jsx
<button onDoubleClick={doubleClick}>
```

Runs only when the button is double-clicked.

### Interview Answer

> `onClick` fires on a single click, whereas `onDoubleClick` fires only after two consecutive clicks.

# 13. What is an inline event handler?

Example

```jsx
<button onClick={() => alert("Hello")}>
```

### Drawback

A new function is created on every render.

### Better

```jsx
const handleClick = () => {};

<button onClick={handleClick}>
```

### Interview Answer

> An inline event handler is an arrow function written directly inside JSX. It is convenient but creates a new function on every render, so a separate handler is preferred when possible.

---

# 14. Can we have multiple event handlers?

Yes.

Example

```jsx
<input
    onFocus={focus}
    onBlur={blur}
    onChange={change}
/>
```

### Interview Answer

> Yes. A component can handle multiple events such as `onFocus`, `onBlur`, and `onChange` simultaneously.

---

# 15. Why do event handlers use arrow functions?

Example

```jsx
<button onClick={() => save(id)}>
```

### Reason

To pass parameters without executing the function immediately.

### Interview Answer

> Arrow functions are commonly used in event handlers to pass arguments or execute additional logic while ensuring the function runs only when the event occurs.

---

# 16. Can event handlers update state?

Yes.

Example

```jsx
const [count, setCount] = useState(0);

<button onClick={() => setCount(count + 1)}>
    Increment
</button>
```

### Interview Answer

> Yes. Event handlers commonly update state using the state setter function. Updating state causes React to re-render the component with the new data.

---

# 17. What is the difference between `onInput` and `onChange`?

### `onInput`

* Fires immediately whenever the input changes.

### `onChange` (React)

* Also fires on each value change for text inputs (unlike plain HTML where `change` often fires on blur).

### Interview Answer

> In React, `onChange` behaves similarly to the browser's `input` event for text fields, firing on every value change. `onInput` is available but is used less frequently.
